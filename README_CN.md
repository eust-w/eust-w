我主要做把 AI 放进真实工程流里的工具：能理解和修改代码库的 agent、围绕云和开发者操作的自动化，以及能减少重复劳动的小工具。

### 当前关注

- Agent-native 软件工程：Codex 工作流、多仓库能力搜索、自动验证和基于证据的代码修改。
- 实用 AI 工具：模型路由、AI 代码审查、LLM 实验，以及尽量贴近可运行产品的开发者工作流。
- 开发者基础设施：用 Go 和 TypeScript 做 SSH、虚拟机镜像、云操作和知识工作流相关 CLI。

### 做事方式

- 优先做能跑起来的系统，而不是只停留在 demo。
- 自动化要可观察：日志、产物、失败原因和可复现步骤都很重要。
- 用小工具积累成更大的工作流。
